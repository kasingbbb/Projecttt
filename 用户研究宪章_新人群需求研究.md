# 用户研究宪章：新人群需求研究

## 1. 项目概述

### 1.1 项目名称
新人群需求探索与研究项目

### 1.2 项目目标
- 识别并评估潜在新用户群体
- 挖掘未被满足的核心需求
- 建立系统化的需求验证框架
- 为产品创新提供数据支持

### 1.3 研究周期
- 探索阶段：2-3周
- 验证阶段：3-4周
- 迭代阶段：持续进行

## 2. 研究框架与方法论

### 2.1 人群选择框架（4P模型）

#### 2.1.1 潜在规模（Potential）
- **评估指标**：
  - 人群基数（绝对数量）
  - 增长率（年/季度）
  - 可触达性（渠道覆盖）
  - 市场渗透率现状
- **数据来源**：
  - 行业报告
  - 人口统计数据
  - 社交媒体活跃度
  - 搜索指数趋势

#### 2.1.2 支付意愿（Payment）
- **评估指标**：
  - 平均消费能力
  - 付费习惯（订阅/一次性）
  - 生命周期价值（LTV）
  - 价格敏感度
- **数据来源**：
  - 消费调研
  - 竞品定价分析
  - 支付行为数据
  - 经济水平统计

#### 2.1.3 痛点强度（Pain）
- **评估指标**：
  - 问题频率（日/周/月）
  - 问题严重程度（1-10分）
  - 现有解决方案满意度
  - 情绪反应强度
- **数据来源**：
  - 用户访谈
  - 社交媒体抱怨
  - 客服反馈
  - 竞品评论分析

#### 2.1.4 可塑性（Plasticity）
- **评估指标**：
  - 技术接受度
  - 使用习惯改变意愿
  - 学习成本接受度
  - 创新意识水平
- **数据来源**：
  - 技术采用曲线分析
  - 行为改变历史
  - 教育背景分布
  - 创新产品使用情况

### 2.2 价值判断标准（5C模型）

#### 2.2.1 市场验证度（Confirmation）
- **验证信号**：
  - 社交媒体主动讨论
  - 早期竞品尝试
  - 行业报告提及
  - 投资趋势指向
- **判断标准**：至少2个强信号

#### 2.2.2 价值密度（Concentration）
- **计算公式**：
  ```
  价值密度 = (单用户潜在价值 × 人群规模) / (开发成本 × 3)
  ```
- **合格标准**：> 1.0

#### 2.2.3 触达成本（Cost）
- **计算公式**：
  ```
  CAC（获客成本）< LTV（生命周期价值）/ 3
  ```
- **数据来源**：相似市场类比分析

#### 2.2.4 竞争格局（Competition）
- **评估维度**：
  - 市场集中度（CR3）
  - 差异化机会空间
  - 进入壁垒高度
  - 替代品威胁程度
- **合格标准**：存在差异化机会且进入壁垒可接受

#### 2.2.5 技术可行性（Capability）
- **评估维度**：
  - 技术成熟度（TRL等级）
  - 团队技术匹配度
  - 开发资源需求
  - 时间可行性
- **合格标准**：TRL ≥ 6，团队匹配度 ≥ 70%

## 3. 需求挖掘流程

### 3.1 四层漏斗挖掘法

#### 第一层：现象收集（宽口径）
- **方法**：
  1. 用户访谈（30-50人，分层抽样）
  2. 行为观察（场景记录，时长≥20小时）
  3. 数据埋点分析（如有）
  4. 竞品功能矩阵对比
- **产出**：原始需求池（200+条）

#### 第二层：需求提炼（结构化）
- **框架应用**：
  1. **Jobs-to-be-Done**：识别用户"雇佣"产品的真实工作
  2. **马斯洛需求分层**：从功能到自我实现的映射
  3. **Kano模型分类**：基本型/期望型/兴奋型/无差异/反向需求
- **产出**：结构化需求清单（50-80条）

#### 第三层：需求验证（科学化）
- **定量方法**：
  - 问卷调研（N≥300，置信度95%）
  - A/B测试（显著性p<0.05）
  - 联合分析（属性重要性排序）
- **定性方法**：
  - 深度访谈（8-12人）
  - 可用性测试（任务完成率）
  - 日记研究（行为模式识别）
- **产出**：验证后的需求优先级

#### 第四层：优先级排序（决策化）
- **排序模型**：
  1. **RICE评分**：覆盖范围×影响力×信心度÷投入成本
  2. **价值/复杂度矩阵**：四象限定位
  3. **KANO验证结果**：优先满足基本型，重点打造兴奋型
- **产出**：需求路线图（季度规划）

### 3.2 需求准确性验证标准

#### 3.2.1 一致性（Consistency）
- **标准**：不同数据源结论一致率 ≥ 80%
- **检查点**：访谈 vs 问卷 vs 行为数据

#### 3.2.2 可操作性（Actionability）
- **标准**：需求可转化为具体功能点（≥3个）
- **检查点**：功能描述清晰度、技术实现路径

#### 3.2.3 可验证性（Verifiability）
- **标准**：有明确的成功指标（≥3个）
- **检查点**：KPI定义、测量方法、目标值

#### 3.2.4 差异性（Differentiation）
- **标准**：与竞品解决方案差异度 ≥ 30%
- **检查点**：功能对比矩阵、用户感知测试

#### 3.2.5 持续性（Sustainability）
- **标准**：需求预计持续时长 ≥ 2年
- **检查点**：趋势分析、用户行为惯性

## 4. MI（市场洞察）研究框架

### 4.1 宏观层：PESTEL分析
- **政策（Political）**：监管环境、行业标准、政策支持
- **经济（Economic）**：消费趋势、购买力、经济周期
- **社会（Social）**：文化变迁、价值观、生活方式
- **技术（Technological）**：技术成熟度、创新周期、技术采用
- **环境（Environmental）**：可持续性、环保要求
- **法律（Legal）**：合规风险、知识产权

### 4.2 中观层：竞争分析
- **波特五力模型**：
  1. 供应商议价能力
  2. 购买者议价能力
  3. 新进入者威胁
  4. 替代品威胁
  5. 同业竞争强度
- **竞品功能矩阵**：50+功能点对比
- **价值链分析**：识别价值洼地

### 4.3 微观层：用户行为分析
- **使用场景地图**：时间×地点×设备×任务流
- **决策旅程分析**：认知→考虑→决策→使用→忠诚
- **触点体验分析**：各接触点的满意度和痛点

## 5. CI（客户洞察）研究框架

### 5.1 定性研究（深度理解Why）
- **情景访谈**：实际场景，时长60-90分钟
- **日记研究**：7天记录，每日回顾
- **影随观察**：全程跟随，时长≥4小时
- **共创工作坊**：8-10人，时长3小时

### 5.2 定量研究（验证What和How Much）
- **大规模问卷**：N≥500，分层抽样
- **眼动实验**：关注点分布、热力图
- **A/B测试**：方案对比，显著性检验
- **支付意愿测试**：价格敏感度分析

### 5.3 混合研究方法
- **顺序混合**：定性→定量→定性
- **并行混合**：同时进行，数据三角验证
- **转化混合**：定性发现转化为定量假设

## 6. 执行流程与时间规划

### 6.1 阶段一：探索发现（2-3周）
- **第1周**：桌面研究、行业扫描
- **第2周**：用户招募、访谈提纲设计
- **第3周**：深度访谈、观察研究
- **产出**：初步洞察报告、假设清单

### 6.2 阶段二：定义聚焦（2周）
- **第4周**：数据分析、模式识别
- **第5周**：用户画像创建、需求提炼
- **产出**：用户画像、需求矩阵、机会点地图

### 6.3 阶段三：验证优化（3-4周）
- **第6-7周**：问卷设计、大规模调研
- **第8周**：概念测试、可用性测试
- **第9周**：数据分析、结论提炼
- **产出**：验证报告、需求优先级、产品建议

### 6.4 阶段四：迭代监控（持续）
- **每月**：市场反馈监测
- **每季度**：用户满意度跟踪
- **每年**：需求演进分析
- **产出**：迭代建议、趋势报告

## 7. 产出物标准

### 7.1 文档标准
- **格式**：Markdown + 可视化图表
- **长度**：执行摘要（1页）+ 详细报告（≤20页）
- **语言**：中文为主，关键术语标注英文
- **版本控制**：Git管理，每次修改记录

### 7.2 数据标准
- **原始数据**：匿名化处理，加密存储
- **分析数据**：可追溯、可复现
- **可视化**：图表清晰，标注完整
- **元数据**：研究时间、样本特征、方法说明

### 7.3 交付物清单
1. 研究计划书
2. 访谈提纲与记录
3. 问卷设计与数据
4. 数据分析报告
5. 用户画像卡片
6. 需求优先级矩阵
7. 产品建议书
8. 执行摘要（管理层版）

## 8. 质量保证机制

### 8.1 研究设计审查
- **审查点**：抽样方法、问卷设计、访谈提纲
- **审查人**：研究负责人 + 方法论专家
- **频率**：每个阶段开始前

### 8.2 数据质量检查
- **检查项**：完整性、一致性、准确性
- **检查方法**：逻辑校验、异常值检测
- **频率**：数据收集后24小时内

### 8.3 结论验证
- **验证方法**：三角验证、专家评审、用户反馈
- **验证标准**：至少2种方法支持结论
- **频率**：报告初稿完成后

## 9. 伦理与合规

### 9.1 用户隐私保护
- **知情同意**：明确告知研究目的
- **数据匿名**：去除个人标识信息
- **数据安全**：加密存储，访问控制
- **数据保留**：研究结束后6个月删除

### 9.2 研究伦理
- **无伤害原则**：不造成心理或生理伤害
- **自愿原则**：随时可退出
- **公平原则**：样本代表性
- **透明原则**：研究过程可追溯
