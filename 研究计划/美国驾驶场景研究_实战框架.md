# 美国驾驶场景研究 - 实战框架

**研究主题：** 美国司机在驾驶过程中的设备使用行为与未被满足的需求

**核心问题：** 他们真的在用车机吗？为什么选择手机？背后的真实原因是什么？

---

## 🎯 研究目标（1句话版）

**发现驾驶场景中被忽视的痛点，找到用户宁愿冒险也要解决的问题。**

---

## 👥 我们要访谈谁？（7-10人）

### 必须访谈的3类人

**类型1：最近切换者（3-4人）⭐⭐⭐**
```
寻找对象：
- 最近6个月从车机切换到手机（或反之）
- 最近开始使用CarPlay/Android Auto
- 最近停止使用某个驾驶App

为什么重要：
"切换"意味着强烈的pain point
他们清楚记得"为什么改变"

招募话术：
"我在找最近改变了开车习惯的人，
 比如开始或停止使用某个App/功能。
 你最近6个月有类似的经历吗？"
```

**类型2：流失用户（2-3人）⭐⭐⭐**
```
寻找对象：
- 下载了驾驶类App但卸载了
- 试用过CarPlay但关闭了
- 买了车载设备但不用了

为什么重要：
最关键的洞察来自"为什么放弃"

招募话术：
"我在研究为什么有些驾驶功能大家不用了。
 你有没有试过某个功能但后来停止使用的经历？"
```

**类型3：极端用户（2-3人）⭐⭐**
```
两种极端：
A. 重度用户（Uber/Lyft司机，长途通勤）
   - 建立了复杂的驾驶工作流
   - 能揭示专业场景需求

B. 零技术用户（从不用导航/手机）
   - 能揭示认知差距
   - "为什么不需要？"

招募标准：
- 重度：每周开车>20小时
- 零技术：从不使用导航App
```

### 招募渠道

**线上：**
- Reddit: r/AskAnAmerican, r/cars, r/driving
- Facebook Groups: "American Drivers", 地方通勤群
- Twitter: #commute, #驾驶相关话题

**线下：**
- 停车场随机招募
- 通勤族密集区（高峰期）
- Uber/Lyft司机群体

**激励：**
- $50-100/小时（美国标准）
- 或Amazon礼品卡

**预期拒绝率：** 90%（正常）
**目标：** 从100个联系中找到7-10个合格的

## 💬 访谈框架（60-75分钟）

### 不要带问题清单，带心理框架

**Four Forces Framework（Bob Moesta）**

在脑海中思考这4个力：
```
1. Push（推力）：
   什么push他们想改变当前的做法？
   - 具体的痛点是什么？
   - 什么时刻特别难受？

2. Pull（拉力）：
   新方案吸引他们什么？
   - 他们期待什么好处？
   - 听谁推荐的？

3. Anxiety（焦虑）：
   什么让他们犹豫？
   - 担心什么风险？
   - 什么阻止他们行动？

4. Habit（习惯）：
   什么让他们保持现状？
   - 已经习惯了什么？
   - 改变的成本是什么？
```

**这不是问卷，这是你的思考框架。**
根据对话自然地探索这4个维度。

---

### 访谈流程

**阶段1：暖场（5分钟）**
```
"感谢你的时间！我在研究美国人开车的真实体验。
 我想听你的故事，没有对错答案。
 我不是来推销任何东西的，纯粹是学习。"

第一个问题（让他们进入回忆）：
"告诉我你昨天（或上次）开车的情况。
 从准备出门开始，一直到到达目的地。"

注意：
- 不要打断
- 让他们自由讲
- 观察他们主动提到什么
```

**阶段2：深挖关键时刻（40-50分钟）**

**核心提问法：时间旅行**
```
"带我回到[那个时刻]..."
"假装我坐在你副驾驶，描述给我看..."
"那个时候你在想什么？"
```

**必须探索的4个场景（选择他们经常遇到的）：**

**场景1：通勤/日常驾驶**
```
核心问题：
"告诉我上次通勤的情况。
 - 出门前你做了什么准备？
 - 开车过程中发生了什么？
 - 有什么事打断了你？"

追问（如果他们提到设备）：
"你什么时候会看手机/车机？"
"给我演示一下你当时怎么操作的。"
"那个时候你的感觉是什么？"

注意观察：
- 是否提到"危险"或"担心"
- 是否有"不得不"的表达
- 情绪词汇（烦躁、焦虑、放松）
```

**场景2：遇到突发情况**
```
核心问题：
"告诉我一次开车时遇到意外的经历。
 - 比如突然的堵车、绕路、紧急电话
 - 当时你怎么处理的？
 - 你用了什么工具或设备？"

深挖：
"为什么那个时候选择[X]而不是[Y]？"
"如果让你重来，你会做什么不同的？"
"最让你焦虑的是什么？"
```

**场景3：长途驾驶**
```
（如果他们有长途经验）
"告诉我上次长途驾驶（>2小时）的情况。
 - 你怎么准备的？
 - 路上发生了什么？
 - 什么让时间过得快/慢？"

重点探索：
- 娱乐内容选择
- 疲劳应对
- 无聊应对
- 中途休息决策
```

**场景4：停车/到达后**
```
"到达目的地后，你通常做什么？"
"有没有需要在车上完成但没时间做的事？"
"停车这件事本身有什么麻烦吗？"
```

**特殊探索：切换时刻**
```
如果是"切换者"，必须问：
"告诉我你决定改变的那一刻。
 - 具体发生了什么事？
 - 是什么压垮骆驼的最后一根稻草？
 - 你尝试了什么新方案？
 - 现在回头看，值得吗？"

如果是"流失者"，必须问：
"告诉我你决定停止使用的那一刻。
 - 什么让你放弃了？
 - 你期待的是什么，实际得到的是什么？
 - 现在用什么替代？"
```

---

**阶段3：演示环节（15分钟）**
```
"现在能不能给我演示一下？
 假装你要开始一段开车，
 展示给我看你通常怎么做。"

观察清单：
- □ 先连接什么设备
- □ 打开哪些App（顺序）
- □ 放在哪里（手机位置）
- □ 触摸屏 vs 语音 vs 物理按键
- □ 设置过程中的停顿（说明思考/犹豫）
- □ 情绪变化（皱眉、叹气、咯咯笑）

关键问题：
"为什么你先做X？"
"这个步骤总是这么麻烦吗？"
"你有没有更快的办法？"
```

---

**阶段4：情感挖掘（10分钟）**
```
Nan Yu原则：
"我的目标是用户怎么难受，我也怎么难受。"

核心问题：
"开车时使用设备，你最大的感受是什么？"
"有没有让你感到内疚或不安的时刻？"
"理想状态下，你希望开车时的体验是什么样的？"
"如果有魔法棒，你会改变什么？"

注意捕捉：
- 负面情绪词（恐惧、焦虑、愤怒）
- 冲突感（"我知道不应该...但是..."）
- 理想与现实的gap
```

---

**阶段5：支付意愿测试（5分钟）**
```
"基于你刚才说的痛点，
 如果有一个解决方案，
 你愿意为它支付多少？"

（等待回答）

进阶版（真实性测试）：
"如果我今天能解决[他们最大的痛点]，
 你愿意现在就支付$20预订早期访问吗？
 我可以接受Venmo/PayPal。"

为什么这么做：
Jeff Weinstein："'愿意付费'和'真的付费'差距巨大。
                立即行动测试真实承诺。"

判断标准：
- 10人中≥3人立即愿意付款 = 强痛点
- 犹豫或降价 = 痛点不够强
- 直接拒绝 = 伪需求
```

---

**结尾（5分钟）**
```
"还有什么我应该问但没问到的？"
（往往有惊喜）

"你认识其他[符合条件]的人吗？
 能帮我介绍吗？"
（滚雪球招募）

"再次感谢！我会整理洞察后分享给你。"
```

## 📊 快速分析（2天）

### Day 1：提取原始洞察

**工具：** Excel/Notion + 便利贴

**步骤：**

**1. 听录音/看笔记，标记关键时刻（3小时）**
```
使用emoji标记：
😮 惊讶时刻："我没想到会这样..."
😡 愤怒时刻："这个太烦人了..."
😰 焦虑时刻："我担心会..."
💡 顿悟时刻："原来是因为..."
🔁 重复模式："又有人提到了..."
```

**2. 创建关键引语库（2小时）**
```
选出10-15条"原话"：
- 特别生动
- 揭示情感
- 代表典型观点

格式：
"[原话]" - [用户类型], [场景]

示例：
"Using my car's screen feels like trying to text
 on a Nokia while everyone else has an iPhone."
 - Uber司机, 城市驾驶
```

**3. 痛点强度排序（1小时）**
```
指标：
- 频率：几个人提到？
- 情绪：情绪强度如何（1-10）
- 行动：有人因此改变行为吗？
- 支付：愿意付费解决吗？

列出Top 5痛点
```

### Day 2：聚类与可视化

**4. 行为聚类（白板会议，2小时）**
```
参与者：PM + 工程师 + 设计师

方法：
1. 每人列出访谈中最深刻的5个发现
2. 便利贴贴到白板上
3. 一起移动，形成主题群
4. 给每个群命名
5. 投票选出Top 3

产出：3-5个用户类型

示例聚类（驾驶场景）：
1. "焦虑的连接者"（Anxious Connectors）
   - 痛点：必须连接，但怕分心
   - 场景：接送孩子的父母
   - 引语："I need to know if school called,
           but I'm terrified of looking at my phone."

2. "效率狂人"（Productivity Junkies）
   - 痛点：开车=浪费时间
   - 场景：长途通勤的销售/顾问
   - 引语："My car is my second office.
           I can't afford to just 'drive'."

3. "无聊战士"（Boredom Fighters）
   - 痛点：驾驶太无聊
   - 场景：Uber司机、长途司机
   - 引语："8 hours on the road, I need something
           to keep my brain alive."

4. "技术抵抗者"（Tech Resistors）
   - 痛点：新功能太复杂
   - 场景：年长司机、简单需求
   - 引语："I just want to get from A to B.
           Why does everything need an app?"
```

**5. 创建情境画像（2小时）**
```
不是传统的人口统计画像，而是"情境画像"：

模板：
【情境】：什么时候遇到问题
【任务】：想完成什么
【痛点】：什么阻碍了
【当前方案】：怎么凑合

【支付意愿】：$X-Y
【关键引语】："..."

```

1页纸，包含：

## 🚨 常见陷阱（驾驶场景特有）

### 陷阱1：过度关注安全
```
❌ 问题："你开车时用手机安全吗？"
   （用户会给社会期待的答案："当然不安全"）

✅ 问题："告诉我上次开车时用手机的情况。"
   （让他们描述真实行为，而非理想行为）

关键：
不是问"应该怎样"，而是观察"实际怎样"
```

### 陷阱2：忽视语音的局限
```
很多人说"我用语音"，但实际：
- 语音识别失败率
- 复杂操作无法语音
- 隐私场景不适合语音

必须追问：
"给我演示一下你上次用语音的情况。"
"有没有失败的时候？那时你怎么办？"
```

### 陷阱3：车机 vs 手机的表面答案
```
用户说："车机不好用"
❌ 接受答案
✅ 深挖："具体哪里不好用？给我演示一次。"

往往发现：
- 不是"不好用"，是"不知道怎么用"
- 不是"功能少"，是"找不到功能"
- 不是"慢"，是"操作路径长"
```

### 陷阱4：假设都是通勤
```
不要只研究上下班通勤
也要探索：
- 周末出游
- 接送家人
- 商务拜访
- 跑腿办事

不同场景，痛点完全不同
```

## ✅ 成功标准

### 研究质量

**你知道做得好，如果：**
- [ ] 至少3个让你惊讶的发现
- [ ] 团队不看文档能说出核心洞察
- [ ] 有至少5条生动的用户原话
- [ ] 工程师说"这改变了我的看法"
- [ ] 有可以立即测试的产品idea

**你知道做得不好，如果：**
- ❌ 所有发现都是"意料之中"
- ❌ 洞察都是"用户想要更好的体验"这种废话
- ❌ 无法回答"为什么用户这么做"
- ❌ 团队讨论时还在争论基本事实
